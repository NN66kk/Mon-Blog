# CodexView：在 macOS 菜单栏和 Codex 标题栏查看额度

使用 Codex 处理稍长一些的任务时，额度往往是一个容易被忽略、又不得不关心的问题：当前额度还剩多少？周额度是否接近上限？下一次重置是什么时候？

这些信息并不是查不到，但如果每次都要离开正在进行的工作、打开其他页面或执行额外命令，原本连贯的节奏就会被打断。

CodexView 就是为此开发的一款 macOS 小工具。它的目标很简单：把 Codex 额度放到随时可见的位置，让用户不必切换页面，也能了解当前的额度状态。

目前，CodexView 已发布首个公开测试版 `v0.1.0-beta`。

## CodexView 是什么

CodexView 是一款原生 macOS 菜单栏应用。启动后，它不会占用 Dock，也不会打开需要额外管理的主窗口，而是在屏幕右上角显示简洁的额度状态。

菜单栏中的 `CV` 状态会直接展示主额度的剩余百分比。展开菜单后，还可以查看：

- 主额度和周额度的剩余情况；
- 距离下一次额度重置的时间；
- 数据最后更新时间及是否已经过期；
- Codex 本地服务的连接状态。

如果需要获取最新数据，也可以通过“立即刷新”手动触发更新。

CodexView 并不试图成为一套复杂的效率工具。它只处理一个明确的问题：减少查看额度所需的操作和注意力切换。

## 在 Codex 标题栏中显示额度

除了菜单栏，CodexView 还可以将一个轻量的 Overlay 显示在 Codex 窗口标题栏的安全区域。

这样一来，无论是在编写代码、查看变更，还是等待任务执行完成，都可以直接在当前窗口中确认额度状态，不需要把注意力转移到其他位置。

这个 Overlay 并不是固定覆盖在窗口表面。CodexView 会结合窗口位置和系统状态判断它是否适合显示。在以下情况下，Overlay 会自动隐藏：

- Codex 不在前台；
- Codex 窗口已最小化；
- 当前处于不适合放置 Overlay 的全屏布局；
- 额度数据已经过期；
- 标题栏没有足够的安全显示空间。

这种设计是为了避免额度提示干扰窗口操作。应用同时支持跟随系统、浅色和深色三种外观，并可以设置为登录时自动启动。

## 辅助功能权限的用途

CodexView 的菜单栏功能不需要额外权限。只有启用标题栏 Overlay 时，应用才需要 macOS 的“辅助功能”权限。

这项权限用于获取 Codex 窗口的位置、大小、最小化状态和全屏状态，以便将 Overlay 放置在合适的位置。CodexView 不会通过该权限读取窗口正文、键盘输入、剪贴板内容或屏幕图像。

用户可以拒绝授权，也可以随时在系统设置中撤销权限。撤销后，标题栏 Overlay 将停止工作，但菜单栏中的额度显示不会受到影响。

## 额度数据从哪里来

CodexView 不读取浏览器 Cookie，不抓取私有网页接口，也不会向 Codex Desktop 注入代码。

它通过本机已经安装并登录的 `codex app-server` 获取额度状态，并且只提取界面展示所需的字段，包括主额度、周额度、重置时间和连接状态。

应用本身没有账号系统，也不包含遥测、广告、云同步或远程日志。完整账号对象、认证令牌和原始响应不会被保存。最近一次有效额度只保留在当前进程的内存中，退出应用后不会持久化。

换句话说，额度数据始终留在本机，CodexView 只是将 Codex 已经提供的本地信息以更方便的方式展示出来。

## 下载与安装

CodexView `v0.1.0-beta` 可以从 GitHub Releases 下载：

[下载 CodexView v0.1.0-beta](https://github.com/NN66kk/codexview/releases/tag/v0.1.0-beta)

在 Release 附件中选择：

`CodexView-v0.1.0-beta-macos-arm64.app.zip`

安装步骤如下：

1. 下载 ZIP 文件并双击解压；
2. 将 `CodexView.app` 拖入“应用程序”文件夹；
3. 打开 CodexView；
4. 在屏幕右上角查看 `CV --` 或额度百分比。

当前版本的运行要求是：

- macOS 14 或更高版本；
- 配备 Apple Silicon 芯片的 Mac；
- 已安装并登录 Codex Desktop。

`v0.1.0-beta` 暂不支持 Intel Mac。

## 首次打开时的 macOS 安全提示

当前测试版尚未经过 Apple 公证，使用的是 ad-hoc 临时签名。因此，第一次启动时，macOS 可能会提示无法验证开发者，并阻止应用直接打开。

遇到这种情况，不需要执行终端命令，可以通过系统设置手动放行：

1. 先尝试打开一次 `CodexView.app`；
2. 打开“系统设置”；
3. 进入“隐私与安全性”；
4. 在安全性区域找到刚刚被阻止的 CodexView；
5. 点击“仍要打开”，并再次确认。

建议只从项目的 GitHub Releases 页面下载安装包。需要进一步核对文件完整性时，可以使用 Release 中提供的 SHA-256 校验文件。

## 为什么先发布 Beta 版本

CodexView 目前已经具备可运行的 App 构建、核心协议检查和 GitHub CI，但距离适合所有普通用户的正式版本仍有一些工作需要完成。

首先，正式分发的 macOS 应用通常需要 Developer ID 签名、Hardened Runtime 和 Apple 公证，以减少安装过程中的安全提示。其次，多显示器、不同 Space、全屏模式、权限撤销、睡眠唤醒和长时间运行等场景，也需要在更多真实设备上验证。

因此，`v0.1.0-beta` 的定位是公开测试版：它已经可以直接使用，但仍然明确保留 Beta 标签，以如实反映当前的分发方式和验证范围。

后续计划包括：

- 完成 Developer ID 签名和 Apple 公证；
- 提供更顺畅的正式安装包；
- 完善多显示器、Space、全屏及长期运行测试；
- 根据实际反馈继续优化兼容性和显示细节。

## 总结

CodexView 解决的是一个很小但具体的问题：让 Codex 用户更方便地掌握额度状态。

它不会增加新的工作流程，也不要求用户频繁打开另一个界面。菜单栏提供随时可见的额度概览，标题栏 Overlay 则让信息直接出现在当前工作环境中。与此同时，应用只从本机 Codex 服务读取必要数据，不保存认证信息，也不依赖远程账号系统。
