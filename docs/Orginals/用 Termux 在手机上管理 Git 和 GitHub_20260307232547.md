# 用 Termux 在手机上管理 Git 和 GitHub

很多人一听到 Git、GitHub、命令行，就会觉得这些东西只能在电脑上操作。其实并不是这样。只要装好 **Termux**，安卓手机也能完成很多基础的 Git 操作，比如下载仓库、修改文件、提交内容、同步到 GitHub。

如果你平时有写博客、记笔记、维护 Obsidian 知识库的需求，那么学会这一套会很方便。这篇文章不讲太多抽象原理，只讲新手真正会遇到的几个问题：Termux 是什么、怎么进入手机文件夹、怎么配置 Git、怎么连接 GitHub，以及遇到报错时该怎么判断。

---

## Termux 是什么

可以把 Termux 理解成手机里的一个“终端窗口”。平时我们在手机上操作文件，靠的是点来点去；而在 Termux 里，主要靠输入命令。

刚开始看上去会有点陌生，因为它没有普通 App 那种按钮界面，但实际上常用命令并不多。只要把最基础的几步弄明白，后面很多操作都会很顺。

---

## 第一步：安装 Git

装好 Termux 之后，通常先更新一下环境，然后安装 Git。

```bash
pkg update
pkg upgrade
pkg install git
```

安装完成后，可以用下面这条命令确认有没有装好：

```bash
git --version
```

如果能看到版本号，说明 Git 已经可以用了。

---

## 第二步：让 Termux 访问手机文件

很多人第一次用 Termux 时，都会碰到一个问题：明明手机里有文件夹，但在 Termux 里就是看不到。原因通常不是文件没了，而是 **Termux 还没有获得手机存储权限**。

先执行下面这条命令：

```bash
termux-setup-storage
```

执行后，系统一般会弹出授权提示，点允许即可。

授权完成后，Termux 就可以访问手机共享存储了。常见路径是：

```bash
cd ~/storage/shared
```

如果你想进入 `Documents` 文件夹，可以继续写：

```bash
cd ~/storage/shared/Documents
```

查看当前目录里有什么内容，可以输入：

```bash
ls
```

这里有个很典型的新手误区：文件夹名字一定要和实际名字一致。比如很多手机里是 `Documents`，不是你随手猜的 `document` 或 `documents` 的其他写法。进不去时，先用 `ls` 看看真实名字。

---

## 第三步：配置 Git 身份

Git 安装好以后，最好先配置一下用户名和邮箱。这样以后你提交内容时，Git 才知道这次修改是谁做的。

```bash
git config --global user.name "你的用户名"
git config --global user.email "你的邮箱"
```

配置完可以检查一下：

```bash
git config --list
```

这里的用户名和邮箱，通常建议尽量和 GitHub 保持一致，后面看提交记录会更清楚。

---

## 第四步：把仓库放在哪里比较合适

如果你打算长期用手机管理博客、笔记或者知识库，建议一开始就把仓库放在固定位置，不要到处乱放。

比较省事的做法，是统一放到手机的 `Documents` 目录下。这样路径清楚，自己容易找，Obsidian 一类的软件访问起来也方便。比如以后你可以把博客仓库、笔记仓库都放在这里，管理起来会整齐很多。

---

## 第五步：从 GitHub 下载仓库

进入你准备存放仓库的目录后，就可以把 GitHub 上的仓库拉到本地。

```bash
git clone 仓库地址
```

例如：

```bash
git clone https://github.com/username/repo.git
```

下载完成后，进入仓库文件夹：

```bash
cd repo
```

到这里，仓库就已经真正来到你的手机里了。

---

## 第六步：最常用的 Git 操作

日常最常见的操作其实不多，核心就是这几步：

```bash
git status
git add .
git commit -m "更新说明"
git push
```

它们分别表示：

-   `git status`：查看哪些文件变了
    
-   `git add .`：把修改加入暂存区
    
-   `git commit -m "..."`：提交并写一条说明
    
-   `git push`：把本地修改推送到 GitHub
    

如果别人已经更新了仓库，你想先同步下来，可以用：

```bash
git pull
```

对大多数新手来说，这几条命令已经够覆盖日常使用了。

---

## 为什么 GitHub 不能直接输密码了

这个问题很多人都会遇到。现在 GitHub 在命令行里，通常已经不再支持直接用账号密码认证了，所以你 push 的时候，不能再像以前那样直接输登录密码。

现在更常见的是两种方式：

-   Token
    
-   SSH Key
    

如果你用的是 HTTPS 仓库地址，一般会接触到 Token；如果你用的是 SSH 地址，一般会接触到 SSH Key。

---

## Token 是什么

Token 可以简单理解成 GitHub 专门发给你的一串“替代密码”。在命令行里推送代码时，看起来像是在输密码，实际上这里应该填的是 Token，而不是 GitHub 账户本身的登录密码。

如果你发现自己用户名没错、密码也没错，但就是认证失败，往往就是因为这里输错了内容。

如果不想每次都重新输入，可以保存凭证：

```bash
git config --global credential.helper store
```

这样一般第一次输完，后面就不用反复输入了。

---

## SSH Key 更适合长期使用

如果你后面会经常在手机上用 Git，SSH Key 通常更省心。它的思路不是每次输密码，而是给设备配一把“钥匙”，让 GitHub 认得这台设备。

生成 SSH Key 的命令是：

```bash
ssh-keygen -t ed25519 -C "你的邮箱"
```

生成后查看公钥内容：

```bash
cat ~/.ssh/id_ed25519.pub
```

把输出内容复制到 GitHub 的 SSH Key 设置页面里，保存后，再测试一下连接：

```bash
ssh -T git@github.com
```

如果提示认证成功，后面用 SSH 地址管理仓库时，通常就不用再频繁输入认证信息了。

---

## `.gitignore` 是干什么的

`.gitignore` 的作用，就是告诉 Git：哪些文件不要管，不要提交，不要同步。

这个文件在博客、笔记、Obsidian 仓库里很重要。因为你的目录里往往不只有真正想同步的内容，还有缓存、插件文件、临时文件、日志文件。如果不忽略这些内容，仓库会很乱。

比如有时候你只改了一篇文章，但 Git 却显示一大堆文件都变了，很可能就是忽略规则没写好。

如果你的需求是“只同步指定文件夹和几个文件”，那 `.gitignore` 基本是必须认真写的。

---

## 常见问题怎么理解

Termux 和 Git 刚开始上手时，报错和提示看起来会很多，但并不是每一条英文都代表严重问题。很多时候，只是提醒，不一定真的坏了。

比较常见的几类情况有这些：

**1. 输入命令后出现自动提示**  
这通常不是错误，而是自动补全功能在工作。第一次看会觉得奇怪，其实属于正常现象。

**2. 只改了一点内容，却显示改了很多文件**  
这通常和换行符、文件格式差异有关，不一定说明你真的改了很多东西。

**3. push 时总是让你重新登录**  
大多是认证信息没有保存，或者 Token / SSH 没配好。

**4. 仓库突然变得很奇怪，甚至报损坏**  
这种情况在手机端不是完全不会发生，尤其是同步中断、文件被其他 App 改动时更容易出现。轻微问题还能修，严重的话重新 clone 一份通常更稳。

---

## 怎样才能少出问题

Termux 本身不算难，真正容易出问题的地方，往往是操作习惯不固定。比较实用的建议就三条：

-   仓库尽量放在固定目录，比如 `Documents`
    
-   认证方式尽量固定，不要一会儿用 Token 一会儿用 SSH
    
-   每次操作按固定流程来，比如先 `git status`，再提交，再推送
    

只要习惯稳定下来，很多莫名其妙的问题都会少很多。

---

## 结尾

对新手来说，Termux 最难的不是命令本身，而是刚开始不熟悉这种操作方式。但只要把最基础的几件事弄明白，比如怎么进入目录、怎么配置 Git、怎么连接 GitHub、怎么判断常见报错，后面其实就顺了。

如果你的需求是维护博客、同步笔记、管理自己的知识库，那么手机上的 Termux 已经完全够用了。它不一定能替代电脑，但作为一个随时可用的轻量工具，确实很实用。
